# Direct Resume

Direct Resume turns a work object into a direct return point for the right local AI coding session.

Open a supported issue page in the browser, and Direct Resume shows the local Codex or Claude Code session that was explicitly linked to that work. No digging through `/resume`, session pickers, old terminal tabs, or buried chat history.

## Supported In V0.1

- Drupal.org issue pages, including `https://www.drupal.org/project/<project>/issues/<id>`.
- Jira Cloud issue pages on `*.atlassian.net`, including `/browse/PROS-370` style URLs.
- Local Codex resume actions.
- Local Claude Code resume actions.
- Copy-first resume by default.
- Optional visible terminal launch on macOS through Terminal.app or iTerm.

## Non-Goals

Direct Resume is intentionally not:

- a generic AI memory platform
- an issue planner
- a team sync product
- a cloud agent orchestration layer
- a workspace manager

The wedge is: open the work object, resume the right local session.

## Install

Clone the repo:

```bash
cd ~/dev
git clone https://github.com/scottfalconer/direct-resume.git
cd direct-resume
```

Run the checks if you want to verify the local checkout:

```bash
npm test
npm run check:companion
npm run check:extension
```

Set up extension pairing:

```bash
npm run setup
```

Start the local companion:

```bash
npm start
```

Load the extension:

1. Open Chrome or another Chromium browser.
2. Go to `chrome://extensions`.
3. Enable `Developer mode`.
4. Click `Load unpacked`.
5. Select this repo's `extension` directory.
6. Open a supported Drupal.org or Jira issue page.
7. Paste the one-time pairing token from `npm run setup` when prompted.

## Use

On a supported issue page:

1. If no session is linked, click `Link current session`.
2. Enter `codex` or `claude`.
3. Paste either the session id or a resume command, such as `codex resume <session-id>` or `claude --resume <session-id>`.
4. Enter the local workspace path for that session.
5. Later, open the same issue page and click `Copy resume`.

Direct Resume stores explicit links as local session bindings. Heuristic matches from Codex and Claude history are shown as candidates but are not persisted as authoritative bindings.

## Local Storage

Direct Resume stores machine-local data under:

```bash
~/.direct-resume
```

Important files:

- `config.json`: machine id, API token, pairing token state, and exec settings.
- `bindings.json`: explicit links from work objects to local agent sessions.

The browser extension stores only the local companion API token in Chrome extension storage.

## Security Model

- The companion binds to `127.0.0.1:38551` by default.
- `/health` and `/api/pair` are unauthenticated.
- `/api/resolve`, `/api/link`, and `/api/resume` require the paired local API token.
- Pairing uses a short-lived, single-use token generated by `npm run setup`.
- Connectors canonicalize work objects; they do not emit shell commands.
- `/api/resolve` returns opaque candidate references, not raw resume commands.
- `/api/resume` re-checks candidate liveness and returns a typed action.
- Copy mode is the default.
- Terminal execution is disabled unless explicitly enabled.

## Optional Terminal Launch

Copy mode is safest and works everywhere. If you want Direct Resume to open a visible terminal on macOS:

```bash
DIRECT_RESUME_EXEC=1 DIRECT_RESUME_TERMINAL=terminal npm start
```

For iTerm:

```bash
DIRECT_RESUME_EXEC=1 DIRECT_RESUME_TERMINAL=iterm npm start
```

The extension shows `Open terminal` only when the companion reports that terminal execution is enabled and available.

## Environment Variables

- `DIRECT_RESUME_HOST`: companion host, default `127.0.0.1`.
- `DIRECT_RESUME_PORT`: companion port, default `38551`.
- `DIRECT_RESUME_HOME`: local store path, default `~/.direct-resume`.
- `DIRECT_RESUME_CODEX_HOME`: Codex data path, default `~/.codex`.
- `DIRECT_RESUME_CLAUDE_HOME`: Claude data path, default `~/.claude`.
- `DIRECT_RESUME_EXEC`: set to `1` to enable visible terminal launch.
- `DIRECT_RESUME_TERMINAL`: `terminal` or `iterm`.

## Architecture

The repo is intentionally one project with internal boundaries:

- `extension/`: Chromium extension UI and message bridge.
- `companion/server.js`: localhost HTTP companion.
- `companion/connectors/`: work-object canonicalizers for Drupal.org and Jira.
- `companion/adapters/`: agent adapters for Codex and Claude Code.
- `companion/core/`: work-object model, evidence, ranking, protocol, and Direct Resume service.
- `companion/stores/`: local config and session binding storage.
- `test/`: focused unit and seam tests.

Planning notes live in `projects/` and `notes/`.
