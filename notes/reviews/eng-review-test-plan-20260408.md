# Test Plan
Generated by /plan-eng-review on 2026-04-08

Repo: direct-resume

## Affected Pages/Routes
- Drupal.org issue page — resolve the current page URL into a canonical work object, display no-match / one-match / multi-match states, and enforce copy-vs-exec behavior.
- Jira issue page on `atlassian.net/browse/KEY-123` — canonicalize tenant + key correctly and resolve local matches without colliding with other Jira sites.
- Local companion `POST /api/resolve` — accept `{url, page_title}` only, run connector matching/canonicalization, and return ranked candidates plus evidence.
- Local companion `POST /api/link` — create/update a portable work anchor and a machine-local session binding.
- Local companion `POST /api/resume` — return copy-safe actions always and exec-safe actions only when local exec is enabled and the chosen match is explicit or user-confirmed.
- Local companion `GET /health` — expose capability flags needed by the extension without leaking machine-specific paths.

## Key Interactions to Verify
- Opening a Drupal issue with an explicit local binding shows a primary Resume action and evidence.
- Opening a Jira issue with no local binding shows Link current session and does not invent a confident match.
- Choosing one candidate from a multi-match list can promote that candidate into a machine-local explicit binding.
- Copy action always returns a valid resume command for the chosen adapter.
- Exec action appears only after local opt-in and only for explicit or confirmed bindings.
- Extension offline state shows a safe startup hint and never attempts direct command execution.

## Edge Cases
- Same Jira key on different tenants must produce different canonical IDs.
- Drupal alias URLs (`/node/<id>` vs `/project/.../issues/<id>`) must collapse to one work object.
- Portable work anchor exists, but the local machine has no matching session binding.
- Multiple inferred candidates exist across different repos or agents.
- Local exec is enabled, but no launcher is configured for the selected adapter.
- Invalid or stale machine-local binding points to a missing session.
- Companion rejects requests with missing/invalid install token or disallowed origin.

## Critical Paths
- Open issue page -> extension sends `{url, page_title}` -> companion canonicalizes -> candidates ranked -> user copies or resumes the correct session.
- Open issue page with no explicit link -> user links current session -> portable anchor stored in Beads -> machine-local binding stored locally -> future visits show direct resume.
- Open issue page on a second machine -> shared anchor is present but local binding is absent -> tool falls back to safe candidate suggestions instead of broken direct resume.
